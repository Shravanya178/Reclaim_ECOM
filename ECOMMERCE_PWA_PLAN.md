# ReClaim E-Commerce PWA Conversion Plan

## Executive Summary
Converting ReClaim into a Progressive Web App (PWA) with full e-commerce functionality for sustainable material marketplace. This plan integrates:
- **PWA Features**: Offline support, installability, push notifications
- **E-Commerce Frontend**: Product catalog, cart, checkout, user accounts
- **E-Commerce Backend**: Order management, payment processing, inventory system
- **Admin Dashboard**: Enhanced admin tools for product/order/inventory management

### 2.2 Database Functions & Triggers

```sql
-- Function to automatically create cart for new users
CREATE OR REPLACE FUNCTION create_user_cart()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO carts (user_id) VALUES (NEW.id);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER create_cart_on_signup
AFTER INSERT ON profiles
FOR EACH ROW
EXECUTE FUNCTION create_user_cart();

-- Function to update material stock on order
CREATE OR REPLACE FUNCTION update_material_stock()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = 'confirmed' AND OLD.status = 'pending' THEN
    -- Reduce stock for all items in the order
    UPDATE materials m
    SET stock_quantity = stock_quantity - oi.quantity
    FROM order_items oi
    WHERE oi.order_id = NEW.id AND oi.material_id = m.id;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_stock_on_order_confirm
AFTER UPDATE ON orders
FOR EACH ROW
EXECUTE FUNCTION update_material_stock();

-- Function to log lifecycle state changes
CREATE OR REPLACE FUNCTION log_material_lifecycle()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.lifecycle_state IS DISTINCT FROM OLD.lifecycle_state THEN
    INSERT INTO lifecycle_logs (material_id, state, description)
    VALUES (NEW.id, NEW.lifecycle_state, 'State changed from ' || OLD.lifecycle_state || ' to ' || NEW.lifecycle_state);
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER log_lifecycle_changes
AFTER UPDATE ON materials
FOR EACH ROW
EXECUTE FUNCTION log_material_lifecycle();
```

## APPENDIX B: API ENDPOINTS

### Cart APIs:
- `GET /cart` - Get user cart
- `POST /cart/items` - Add item to cart
- `PUT /cart/items/:id` - Update cart item quantity
- `DELETE /cart/items/:id` - Remove from cart

### Order APIs:
- `POST /orders` - Create order
- `GET /orders` - Get user orders
- `GET /orders/:id` - Get order details
- `PUT /orders/:id/status` - Update order status (admin)
- `POST /orders/:id/cancel` - Cancel order

### Payment APIs:
- `POST /payments/initiate` - Initiate payment
- `POST /payments/verify` - Verify payment
- `POST /payments/refund` - Process refund (admin)

### Inventory APIs (Admin):
- `GET /inventory` - Get inventory report
- `PUT /inventory/:id` - Update stock
- `POST /inventory/bulk-upload` - Bulk upload
